# Спецификация: Гибридная структура данных "Человек-ИИ" (v1.0)

Этот документ описывает стандарт хранения данных в компании Galt, обеспечивающий эффективное взаимодействие людей и ИИ-агентов (Antigravity, Cursor, Copilot).

## 1. Философия системы

Система построена на трех принципах:
1.  **Приватно по умолчанию (Private by Default)**: Все новое считается черновиком и невидимо для глобальной системы, пока не опубликовано.
2.  **Песочница для роботов (AI Sandbox)**: Агенты имеют свое пространство для экспериментов (`_Drafts`), где не могут навредить.
3.  **Гибридный контекст (Context Chain)**: Правила наследуются от папки к папке, а метаданные заполняются автоматически где это возможно.

---

## 2. Структура папок и Песочницы (`_Drafts`)

Для безопасной работы ИИ и людей, каждая функциональная папка должна иметь зону карантина.

### 2.1. Правило `_Drafts`
*   **Что это**: Подпапка `_Drafts` внутри любого раздела (например, `02-HR/_Drafts/`).
*   **Для чего**: Место для черновиков, экспериментов, генерации идей ИИ.
*   **Видимость**: Глобальные агенты (индексаторы) **ИГНОРИРУЮТ** эту папку. Это "слепая зона" для системы.
*   **Формат**: Здесь НЕ требуются метаданные или строгие имена.

### 2.2. Протокол Публикации (Publishing)
Чтобы документ стал "Официальным":
1.  Файл перемещается из `_Drafts` в основную папку (например, в `02-HR/`).
2.  Файлу добавляется **YAML Frontmatter** (см. ниже).
3.  Только после этого ИИ начинает учитывать этот файл как часть базы знаний.

---

## 3. Стандарты Файлов (Metadata)

Все "опубликованные" Markdown-файлы ОБЯЗАНЫ иметь YAML-шапку.

```yaml
---
# === Идентификация ===
id: unique-slug-id          # Уникальное имя (напр. policy-vacation-2026)
type: policy                # policy | contract | report | instruction
status: approved            # draft | review | approved | obsolete

# === Доступ и Владение ===
ai_access: true             # Флаг видимости для ИИ. Нет флага = Игнор.
owner: [Role]               # Владелец бизнес-процесса (напр. HR-Director)
author: [GitUser]           # Технический автор (из Git)
classification: internal    # public | internal | confidential

# === Контекст для ИИ ===
summary: "Политика отпусков на 2026 год, включая удаленную работу."
topics: [hr, vacation, remote]
mentions: [@Levan]          # Связанные люди

# === Жизненный цикл ===
validity: 
  end: 2026-12-31
---
```

### Авто-определение Владельца
Система использует гибридную логику:
1.  **Автор (Author)**: Берется из `git config user.email` (кто создал комит).
2.  **Владелец (Owner)**: Если не указан явно, берется из файла `_AI_CONTEXT.md` текущей папки (см. ниже).

---

## 4. Стандарты Директорий (`_AI_CONTEXT.md`)

Каждая папка должна "объяснять" ИИ, что в ней находится.

**Файл: `_AI_CONTEXT.md`**
```markdown
# Контекст: [Название Папки]

## Назначение
[Описание бизнес-функции этой папки]

## Владение (Default Context)
- **Role**: [Роль владельца, напр. Project Manager]
- **Team**: [Команда]

## Правила для ИИ
- Все новые идеи сохранять в `_Drafts`.
- Имена файлов: `YYYY-MM-DD-Topic`.
```

---

## 5. Универсальный Протокол Агента

Любой новый ИИ-агент (Cursor, Windsurf, Custom) должен быть настроен так:

1.  **INIT**: Прочитать корневой файл `.cursorrules` (или `AI_GUIDELINES.md`).
2.  **IGNORE**: Исключить из контекста все папки `_Drafts` (через `.cursorignore`).
3.  **CONTEXT**: Перед работой в папке — прочитать `_AI_CONTEXT.md`.
4.  **SANDBOX**: Если нужно создать новый контент — создавать его ТОЛЬКО в `_Drafts`, если пользователь не попросил иначе.

---

## 6. Техническая реализация

Файл `.cursorignore`:
```text
**/_Drafts/*
**/_drafts/*
```

Файл `.cursorrules`:
```text
PRIME DIRECTIVE: DO NOT treat files in `_Drafts` as system truth.
ALWAYS read `_AI_CONTEXT.md` before answering questions about a folder.
DEFAULT creation path for new ideas is `./_Drafts/`.
```
