# Спецификация: Гибридная структура данных "Человек-ИИ" (v0.6)

Для обеспечения беспрепятственного сотрудничества между людьми и ИИ-агентами (Antigravity, Cursor и т.д.), все данные в Структуре Управления Galt должны следовать этим рекомендациям.

## 1. Принцип "Карантин Черновиков" (Draft Quarantine)

Вместо сложной системы метаданных для скрытия файлов, мы используем **физическое разделение**.

### Правило
Каждая значимая папка (Департамент или Проект) должна содержать подпапку `_Drafts`.
*   **Любой новый файл** создается человеком внутри `_Drafts`.
*   **Агенты ИГНОРИРУЮТ** всё содержимое папок `_Drafts` (через `.cursorignore` / `.gitignore`).
*   В этой зоне человек может писать без правил, без YAML, экспериментировать и ошибаться.

### Процесс Публикации (The Publishing Flow)
Чтобы файл стал "официальным" и виден ИИ:
1.  Человек перемещает файл из `_Drafts` в основную папку (например, из `02-HR/_Drafts/holiday_ideas.txt` в `02-HR/02-04-Regulations/Policy-Holidays.md`).
2.  В этот момент файл **обязан** получить YAML Frontmatter.

## 2. Техническая реализация игнорирования

В корне репозитория создается файл `.cursorignore` (и `.gitignore`, если черновики не должны идти в облако):

```text
# Ignore all draft folders recursively
**/_Drafts/*
**/_drafts/*
**/draft/*
```

Это гарантирует, что `Cursor` (и другие агенты) физически не увидят эти файлы при индексации кодовой базы.

## 3. Стандарты файлов (Markdown) для опубликованных документов

(Применяются ТОЛЬКО к файлам вне папки `_Drafts`)

```yaml
---
id: unique-slug
type: policy
status: approved
ai_access: true (теперь подразумевается по умолчанию, так как файл в публичной зоне)
summary: "..."
---
```

## 4. Концепция "Цепочка Контекста"
(Без изменений)

## 5. Обновленный Алгоритм Агента

1.  **INIT**: Прочитать `.cursorrules`.
2.  **SCOPE**: Агент видит только "Чистую" структуру (без мусора и черновиков).
3.  **VALIDATE**: Если Агент встречает файл без YAML в *основной* структуре — он должен пометить его как "Ошибка: Неоформленный документ" или предложить переместить его в `_Drafts`.
