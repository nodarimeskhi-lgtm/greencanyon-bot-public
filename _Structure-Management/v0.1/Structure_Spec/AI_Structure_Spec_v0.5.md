## 2. Стандарты файлов (Markdown) - Расширенные метаданные

Мы добавляем поля, которые помогают ИИ лучше индексировать, классифицировать и использовать информацию без необходимости читать весь файл.

```yaml
---
# 1. Идентификация (Обязательно)
id: unique-slug
type: policy | contract | meeting-notes
status: draft | approved

# 2. Управление доступом (Обязательно для ИИ)
ai_access: true   # Если false или отсутствует - ИИ игнорирует файл
owner: [Role]
author: [GitUser]
classification: public | internal | confidential | secret

# 3. Контекст для ИИ (Опционально, но рекомендуется)
summary: "Краткое содержание (1-2 предложения) для экономии токенов при поиске"
topics: [budget, 2024, construction, tsalka]
language: ru

# 4. Взаимосвязи (Граф знаний)
mentions: [@Levan, @CompanyX] # Люди и организации
related_files:
  - path/to/strategy.md
dependencies: 
  - id: parent-policy-v1  # Этот документ зависит от другого

# 5. Жизненный цикл
validity:
  start: 2024-01-01
  end: 2024-12-31
next_review: 2024-06-01
---
```
