# Спецификация: Гибридная структура данных "Человек-ИИ" (v0.7)

Для обеспечения беспрепятственного сотрудничества между людьми и ИИ-агентами (Antigravity, Cursor и т.д.), все данные в Структуре Управления Galt должны следовать этим рекомендациям.

## 1. Принцип "Изоляция Экспериментов" (AI Sandbox)

Мы используем папки `_Drafts` как безопасную песочницу.
**Главное отличие**: ИИ *может и должен* создавать файлы здесь, но эти файлы **не считаются истиной** для системы.

### Правила Игры
1.  **Зона Экспериментов**: В папке `_Drafts` можно создавать альтернативные версии политик, безумные идеи, черновики контрактов.
2.  **Невидимость для Системы**: Глобальные агенты (индексаторы) игнорируют эти папки. Если агент ищет "Действующую Политику Отпусков", он *никогда* не должен смотреть в `_Drafts`.
3.  **Видимость для Сессии**: Текущий агент (с которым вы общаетесь) *видит* эти файлы, если вы явно попросите ("Сравни черновик в _Drafts с оригиналом").

## 2. Протокол Создания (Draft Creation)

Когда вы просите ИИ "Создать черновик новой стратегии":
1.  Агент **ОБЯЗАН** сохранить его в папку `_Drafts/` соответствующего раздела (или создать её).
2.  Файл **НЕ ТРЕБУЕТ** строгих метаданных (YAML), пока он в черновиках.
3.  Имя файла может быть свободным (например, `idea-levan-v1.md`).

## 3. Протокол "Promote to Production" (Публикация)

Процесс превращения эксперимента в правило:
1.  Человек (или ИИ по команде) перемещает файл из `_Drafts/` в корень папки (например, `01-Management/`).
2.  **ОБЯЗАТЕЛЬНОЕ ДЕЙСТВИЕ**: Добавление YAML Frontmatter.
    ```yaml
    ---
    status: approved (или review)
    ai_access: true
    ---
    ```
3.  Только теперь этот файл становится частью базы знаний компании.

## 4. Техническая реализация

Файл `.cursorignore`:
```text
# Ignore content specifically for global indexing contexts
**/_Drafts/*
```

Файл `.cursorrules` (дополнение):
```text
IF User asks to "Generate a plan/doc/idea":
  1. DEFAULT to creating it in `./_Drafts/`.
  2. DO NOT overwrite existing files in main folders.
```
