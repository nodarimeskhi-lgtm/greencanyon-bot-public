# Спецификация: Гибридная структура данных "Человек-ИИ" (v0.2)

Для обеспечения беспрепятственного сотрудничества между людьми и ИИ-агентами (Antigravity, Cursor и т.д.), все данные в Структуре Управления Galt должны следовать этим рекомендациям.

## 1. Принципы

1.  **Самоописание**: Каждый файл и папка должны объяснять свое назначение и контекст.
2.  **Машиночитаемые метаданные**: Ключевые атрибуты (Статус, Владелец, Права ИИ) должны быть в структурированных форматах.
3.  **Гибридное владение**: Сочетание ролевой ответственности (из контекста папки) и авторства (из Git).

## 2. Стандарты файлов (Markdown)

Все файлы Markdown (`.md`) ДОЛЖНЫ включать блок YAML frontmatter.

```yaml
---
id: unique-slug-or-uuid
type: policy | procedure | contract | meeting-notes | project-doc
status: draft | review | approved | obsolete
owner: [Role] (Авто-определяется, см. Раздел 5)
author: [GitUser] (Авто-определяется из Git)
ai_permissions:
  read: true
  edit: suggestion-only | full-access
---
```

## 3. Стандарты директорий (`_AI_CONTEXT.md`)

Каждая значимая папка ДОЛЖНА содержать файл `_AI_CONTEXT.md`.

**Шаблон:**

```markdown
# Контекст: [Имя папки]
## Назначение
[Описание]
## Владельцы (Default Owners)
- **Codeowner**: @marketing-team (Группа в Gitea/GitHub)
- **Role**: CMO (Директор по маркетингу)
```

## 4. Алгоритм определения Владельца и Автора

Система использует гибридный подход для заполнения метаданных, чтобы минимизировать ручной ввод.

### Атрибут `author` (Создатель)
*   **Источник**: Git History.
*   **Логика**: При создании или коммите файла скрипт берет `user.name` и `user.email` из конфига Git.
*   **Пример**: `author: Levan <levad@...>`

### Атрибут `owner` (Ответственный)
*   **Источник**: Каскадное наследование (Waterfall).
*   **Логика**:
    1.  **Явное указание**: Если в YAML файла поле `owner` заполнено вручную -> Использовать его.
    2.  **Контекст папки**: Если нет, брать поле `Role` из файла `_AI_CONTEXT.md` текущей директории.
    3.  **Родитель**: Если нет, подниматься вверх по папкам до `01-Management` и т.д.
    4.  **Фолбэк**: Если нигде не указано -> `owner: Unassigned / Review needed`.

**Пример сценария**:
*   *Levan* (Git User) создает файл в папке `02-HR`.
*   В папке `02-HR/_AI_CONTEXT.md` указано `Role: HR-Director`.
*   **Итог в метаданных**:
    *   `author: Levan` (Технический создатель)
    *   `owner: HR-Director` (Бизнес-владелец)

## 5. Нетекстовые файлы

Использовать `manifest.yaml` для маппинга бинарных файлов.
