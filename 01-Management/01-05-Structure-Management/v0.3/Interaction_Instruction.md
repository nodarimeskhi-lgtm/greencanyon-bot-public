# Инструкция по Взаимодействию (v0.3)

## 1. Протокол Публикации (Publishing Protocol)

Процесс превращения черновика в официальный документ.

1.  **Создание**: Работа ведется в папке `_Drafts`. Там файл может называться как угодно и не иметь структуры.
2.  **Перемещение**: Когда файл готов, он перемещается из `_Drafts` в родительскую папку (на уровень выше).
3.  **Оформление**:
    *   Файлу присваивается правильное имя (см. *Rules_Instruction.md*).
    *   В начало файла добавляется блок YAML Frontmatter со статусом `status: approved` (или `review`) и `ai_access: true`.
4.  **Результат**: С этого момента ИИ-агенты начинают "видеть" этот файл и использовать его как источник правды.

## 2. Протокол Работы ИИ-Агента (Universal Agent Protocol)

Любой ИИ-агент (Antigravity, Cursor, Windsurf, Custom Scripts) обязан следовать алгоритму:

1.  **INIT (Инициализация)**:
    *   Прочитать корневые правила (`.cursorrules` или аналог).
    *   Исключить из индекса все папки, соответствующие паттерну `**/_Drafts/*`.

2.  **CONTEXT (Сбор контекста)**:
    *   Перед выполнением задачи в конкретной папке, агент ОБЯЗАН прочитать файл `_AI_CONTEXT.md` в этой папке (если он есть).
    *   Это дает агенту понимание роли, владельца и специфики данных.

3.  **SANDBOX (Песочница)**:
    *   Если агенту нужно создать новый файл, он **по умолчанию** создает его в подпапке `_Drafts`.
    *   Агент **никогда** не перезаписывает существующие "официальные" файлы без прямого указания пользователя.

4.  **PRIVATE BY DEFAULT (Приватность)**:
    *   Если в метаданных файла (`ai_access`) не указано `true`, агент считает файл приватным и не должен использовать его содержимое для ответов на общие вопросы (если только пользователь явно не открыл этот файл в контексте).

## 3. Взаимодействие Человек-ИИ

*   **Задача**: "Создай отчет".
*   **Действие ИИ**: Создает отчет в `.../_Drafts/Draft-Report.md`.
*   **Задача**: "Опубликуй отчет".
*   **Действие ИИ (или Человека)**:
    1.  Проверяет качество.
    2.  Перемещает файл из `_Drafts`.
    3.  Добавляет YAML header.
    4.  Комитит в Git (если настроено).
