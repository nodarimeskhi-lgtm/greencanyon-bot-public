# Инструкция по Правилам Данных (v0.2)

## 1. Контекст Папки (`_AI_CONTEXT.md`)

Каждая значимая папка должна содержать файл `_AI_CONTEXT.md`. Этот файл объясняет ИИ-агентам назначение папки и правила работы в ней.

**Шаблон `_AI_CONTEXT.md`:**
```markdown
# Контекст: [Название Папки]

## Назначение
[Краткое описание того, что хранится в этой папке и для каких бизнес-процессов это нужно.]

## Владение (Context Owners)
- **Role**: [Роль ответственного сотрудника, например: Главный Архитектор]
- **Team**: [Название отдела/команды]

## Правила для ИИ
- Где создавать черновики: `./_Drafts`
- Специфичные форматы файлов: [напр. только PDF для чертежей]
```

## 2. Именование Файлов

Для поддержания порядка используется стандарт именования.

*   **Дата в начале**: Для документов, привязанных к времени (отчеты, протоколы, письма).
    *   Формат: `YYYY-MM-DD-Descriptive-Name.ext`
    *   Пример: `2026-02-01-Board-Meeting-Minutes.md`
*   **Смысловое имя**: Для справочников, инструкций, политик.
    *   Формат: `Descriptive-Name.ext`
    *   Пример: `Vacation-Policy.md`
*   **Язык**: Предпочтительно английский (транслит допускается, но нежелателен) или русский, если это жесткое требование локального процесса. Избегайте пробелов, используйте дефисы `-`.

## 3. Метаданные Файлов (YAML Frontmatter)

Все "официальные" (опубликованные) Markdown-файлы должны начинаться с блока метаданных YAML. Это позволяет ИИ точно понимать статус и содержание документа.

### 3.1. Обязательные поля

```yaml
---
# 1. Идентификация (Обязательно)
id: unique-slug                 # Уникальный идентификатор (slug)
type: policy | instruction | report | contract | meeting-notes
status: draft | review | approved | obsolete

# 2. Управление доступом (Обязательно для ИИ)
ai_access: true                 # Если false или отсутствует - ИИ игнорирует файл
owner: [Role]                   # Ответственная роль
author: [GitUser]               # Автор документа
classification: public | internal | confidential | secret
---
```

### 3.2. Рекомендуемые поля (Контекст для ИИ)

```yaml
# 3. Контекст для ИИ (Рекомендуется)
summary: "Краткое содержание (1-2 предложения) для экономии токенов при поиске"
topics: [budget, 2024, construction, tsalka]
language: ru | en | ka
```

### 3.3. Опциональные поля (Взаимосвязи и жизненный цикл)

```yaml
# 4. Взаимосвязи (Граф знаний)
mentions: [@Levan, @CompanyX]   # Люди и организации
related_files:
  - path/to/strategy.md
dependencies: 
  - id: parent-policy-v1        # Этот документ зависит от другого

# 5. Жизненный цикл
validity:
  start: 2024-01-01
  end: 2024-12-31
next_review: 2024-06-01
```

## 4. Папки `_Drafts`

*   **Назначение**: "Песочница" для черновиков, временных файлов и генераций ИИ.
*   **Правило**: Файлы в `_Drafts` **не** требуют метаданных и **не** считаются официальной истиной.
*   **Видимость**: Глобальные поисковые агенты игнорируют содержимое `_Drafts`, чтобы не засорять базу знаний неверной информацией.
