# Structure Management v0.4

Добро пожаловать в версию 0.4 системы управления структурой компании Galt.

Эта версия является развитием версии v0.3, обновлённой по результатам глубокого аудита на соответствие теоретическим рамкам Минцберга, PMBOK и Johnny.Decimal.

## Что нового в v0.4

1. **Минцберг**: Расширена модель с 4 до 6 частей организации (добавлены Срединная линия и Идеология). Переклассифицированы `08-Sales-Marketing` (→ Операционное ядро) и `09-IT-Security` (→ Техноструктура).
2. **PMBOK**: Добавлены `1-7-Scope`, `7-Commissioning`, `8-Closing`. Перенесён `Author-Supervision` из `3-Design` в `4-Construction`.
3. **Johnny.Decimal**: Документирован принцип двойной индексации (XX для корпоративного, X для проектного уровня).
4. **Правила**: Расширен шаблон `_AI_CONTEXT.md` (+ Минцберг, координация, права). Расширен YAML `type` до 15 значений. Добавлен раздел кросс-ссылок функция↔проект.
5. **Практика**: Созданы 28 файлов `_AI_CONTEXT.md`, 43 папки `_Drafts`. Исправлены нарушения именования и незадекларированные папки.
6. **Документация**: Создана `_Human_Docs/` с теоретическим руководством, user guide и changelog.

## Состав документации

1.  **[[Structure_Instruction]]** — Структура (Structure)
    *   *Для кого*: Для всех сотрудников и системных администраторов.
    *   *О чем*: Какие папки создавать, где что хранить, иерархия проектов, принцип индексации.

2.  **[[Rules_Instruction]]** — Правила Данных (Rules)
    *   *Для кого*: Для тех, кто создает документы.
    *   *О чем*: Как именовать файлы, какие метаданные (YAML) добавлять, кросс-ссылки, форматы.

3.  **[[Interaction_Instruction]]** — Взаимодействие (Interaction)
    *   *Для кого*: Для пользователей ИИ-агентов и разработчиков промптов.
    *   *О чем*: Как публиковать документы, как работают `_Drafts` и `_AI_CONTEXT.md`.

4.  **[[Human_Behavior_Guide]]** — Гайд для Людей (Human Guide)
    *   *Для кого*: Для людей.
    *   *О чем*: Философия системы, простые привычки для эффективной работы.

5.  **[[Structure_Analysis]]** — Теоретическое обоснование
    *   *Для кого*: Для понимания «почему так устроено».
    *   *О чем*: Минцберг (6 частей), PMBOK, Johnny.Decimal, PARA.

6.  **[[Organizational_Roles]]** — Роли и ответственность
    *   *Для кого*: Для каждого сотрудника.
    *   *О чем*: Кто за что отвечает, матрица «папка → роль → Минцберг».
