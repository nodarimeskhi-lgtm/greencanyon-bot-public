# Green Canyon ტოკენიზაციის ბრიფი (Vendor Brief)

ეს დოკუმენტი განკუთვნილია ბლოკჩეინ-დეველოპმენტ და STO (Security Token Offering) სააგენტოებთან გასაგზავნად (მაგ. Tokens.ge), რათა მივიღოთ პროექტის ზუსტი ტექნიკური შეფასება და განფასება (Quotation/Proposal).

## 1. ვინაობა და პროექტის აღწერა
* **პროექტის სახელი:** Green Canyon Eco Village (წალკა, საქართველო).
* **აქტივის ტიპი:** Premium ეკო-სოფელი და პრემიუმ ინფრასტრუქტურა შერეული დანიშნულებით (აპარტ-ოტელები, დამოუკიდებელი კოტეჯები, მზის პანელები).
* **პროექტის მიზანი:** უძრავი ქონების ან გაქირავებიდან მისაღები შემოსავლების (Cash-flow) ტოკენიზაცია (Security Token). მიზანია კაპიტალის მოზიდვა, საინვესტიციო ბარიერის შემცირება (Fractional Ownership) და ფართო საერთაშორისო ბაზარზე გასვლა.

## 2. ტოკენიზაციის მოდელის სამიზნე პარამეტრები
* **რა ტოკენიზდება?** გადასაწყვეტია: აპარტ-ოტელის კონკრეტული ბლოკი, ექსკლუზიური კოტეჯების ჯგუფი თუ კომპლექსის ეკო-ინფრასტრუქტურის (მზის პანელების) წილი.
* **ტოკენის ბუნება:** Security Token — საინვესტიციო აქტივი, რომელიც მფლობელს ანიჭებს დივიდენდის ან მესაკუთრეობის წილობრივ უფლებას. 
* **დივიდენდის განაწილების პრინციპი:** ავტომატიზირებული განაწილება (მაგ. თვეში/კვარტალში ერთხელ) სტეიბლკოინებში (USDT/USDC). 

## 3. ტექნიკური მოთხოვნები (Technical Scope)
* **ბლოკჩეინის შერჩევა:** ჩვენი მოთხოვნაა იაფი და სწრაფი ქსელი (მაგ. Polygon, Ethereum Layer-2-ები). ტოკენის სტანდარტი უნდა შეესაბამებოდეს RWA (Real World Asset) მოთხოვნებს (მაგ. ERC-3643), რაც KYC-გავლილ მისამართებს შორის ტრანზაქციებს აკონტროლებს.
* **ჭკვიანი კონტრაქტები (Smart Contracts):** ტოკენის ემისიის (Minting), გაფინანსების მართვის და დივიდენდების დისტრიბუციის სკრიპტები. კონტრაქტებმა უნდა გაიარონ საერთაშორისო CyberSecurity აუდიტი (მაგ. CertiK ან Hacken).
* **საინვესტიციო პლატფორმა (Investor Dashboard):**
  * ვებ-პლატფორმა მომხმარებლებისთვის (რეგისტრაცია, პორტფელის ისტორია).
  * KYC / AML ვერიფიკაციის მოდულის ინტეგრაცია (მაგ. Sumsub).
  * Fiat (ბარათით) და Crypto (საფულით) მიღების სისტემების ინტეგრაცია (On-ramp).

## 4. იურიდიული მხარე და (Compliance საქართველოში)
* **მოთხოვნა ვენდორს:** საქართველოს ეროვნული ბანკის (NBG) VASP რეგულაციებისა და "სეკურიტიზაციის შესახებ" ახალი კანონის გათვალისწინებით იურიდიული მოდელის (მაგ. SPV-ის შექმნის) შემოთავაზება და თანმხლები მხარდაჭერა.

## 5. კითხვარი ვენდორს (შემსრულებელს)
1. **გამოცდილება:** გაქვთ თუ არა საქართველოში უძრავი ქონების / რეალური აქტივების ტოკენიზაციის პრაქტიკული გამოცდილება (გთხოვთ გვიჩვენოთ პორტფოლიო)?
2. **სერვისის მოცულობა (Scope):** შეგიძლიათ თუ არა სრული "Turn-key" გადაწყვეტილების მოწოდება (იურიდიული+ტექნიკური), თუ მხოლოდ პლატფორმის დეველოპმენტს უზრუნველყოფთ?
3. **ბიუჯეტირება:** გთხოვთ მოგვაწოდოთ STO პროცესის მიახლოებითი ფინანსური შეფასება (პლატფორმა + Smart contract + Security audit) და მოწოდების დრო (Timeframe).
4. **მარეგულირებელი მხარდაჭერა:** რა სახის იურიდიულ მხარდაჭერას უწევთ გუნდს ტოკენომიკის შემუშავებასა და მარეგულირებლის მოთხოვნების დაკმაყოფილებაში?
