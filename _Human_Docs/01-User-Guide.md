# Руководство пользователя: работа с ИИ-агентами в Galt

> Этот документ — для сотрудников, которые используют Cursor IDE или Antigravity (Gemini) в своей ежедневной работе.

---

## 1. Введение

### Что такое гибридная система Human-AI?

В Galt люди и ИИ-агенты работают **вместе**. ИИ помогает:
- Генерировать черновики документов
- Анализировать данные и находить информацию
- Поддерживать структуру файлов в порядке

Но ИИ **не заменяет** человека. Все важные решения принимает человек. ИИ — это инструмент, как калькулятор или Excel.

### 2-минутный обзор

1. Файлы компании организованы в **пронумерованные папки** (00-09)
2. ИИ-агент знает правила и умеет работать в этой структуре
3. Все новые файлы ИИ создаёт в папке `_Drafts` (черновики)
4. Человек проверяет, утверждает и переносит файл из `_Drafts` на постоянное место

## 3. Правила работы с папками

### 3.1. Где создавать файлы

**Правило #1**: Все новые файлы создаются СНАЧАЛА в `_Drafts`.

```
Правильно: 01-Management/01-03-Reports/_Drafts/Quarterly-Report.md
Неправильно: 01-Management/01-03-Reports/Quarterly-Report.md
```

ИИ-агент делает это автоматически (это называется **Sandbox Protocol**).

### 3.2. Как именовать файлы

**С датой** (для отчётов, протоколов, писем):
```
2026-02-01-Board-Meeting-Minutes.md
2026-01-15-Risk-Assessment-Tsalka.md
```

**Без даты** (для инструкций, политик, шаблонов):
```
Vacation-Policy.md
Contract-Template-Supply.md
```

**Запрещено:**
- Пробелы: `My Report.md` -> `My-Report.md`
- Русские буквы в именах: `Отчёт.md` -> `Report.md` или `Otchyot.md`
- Спецсимволы: `report#1.md` -> `Report-01.md`

### 3.3. Как публиковать документ

1. ИИ создаёт файл в `_Drafts`
2. Вы проверяете содержимое
3. Добавляете YAML-заголовок (метаданные) — ИИ может помочь
4. Перемещаете файл из `_Drafts` в целевую папку
5. Готово — документ «опубликован»

### Что такое YAML-заголовок?

Это блок метаданных в начале файла:

```yaml
---
id: board-minutes-2026-02-01
type: meeting-notes
status: approved
ai_access: true
owner: CEO
author: @levadiani
classification: internal
summary: "Протокол заседания Совета директоров от 1 февраля 2026"
language: ru
---
```

ИИ использует эти метаданные, чтобы понимать, что за документ перед ним.

## 5. Работа с ИИ-агентом

### Как правильно ставить задачи

**Хорошие запросы:**
- *«Создай проект бюджета Tsalka на Q2 2026 на основе данных из 1-3-Budget»*
- *«Найди все документы, связанные с согласованием ТУ на водоснабжение»*
- *«Подготовь черновик протокола совещания от 05.02.2026»*

**Плохие запросы:**
- *«Сделай что-нибудь с финансами»* (слишком размыто)
- *«Удали все старые файлы»* (опасно, ИИ не должен удалять)

### Что ИИ делает автоматически

1. **Sandbox Protocol**: Создаёт файлы ТОЛЬКО в `_Drafts`
2. **Metadata Protocol**: Добавляет YAML-заголовок с указанием модели
3. **Language Protocol**: Отвечает на том языке, на котором вы задали вопрос
4. **Foam Wikilinks**: Добавляет ссылки между связанными документами

### Что ИИ НЕ может делать

- **Утверждать документы** — это делает человек
- **Перемещать файлы из `_Drafts`** в финальные папки — это делает человек
- **Принимать бизнес-решения** — ИИ может подготовить аналитику, но решение за вами
- **Удалять файлы** — только человек решает, что удалять

### Как проверять работу ИИ

1. Прочитайте сгенерированный документ
2. Проверьте факты и цифры (ИИ может ошибаться)
3. Убедитесь, что файл в правильной папке `_Drafts`
4. Проверьте YAML-заголовок (тип, статус, владелец)

## 6. FAQ — Частые вопросы

### «Куда положить договор?»
- **Шаблон** (типовой договор) -> `04-Legal/04-02-Contracts-Templates/`
- **Подписанный договор по проекту** -> `05-Projects/Tsalka/2-Permits-Legal/2-4-Project-Contracts/`

### «ИИ создал файл не в той папке, что делать?»
1. Переместите файл вручную в правильную папку `_Drafts`
2. Сообщите ИИ, что папка была неверной — он запомнит для будущих задач

### «Как найти все документы по проекту Tsalka?»
1. Откройте папку `05-Projects/Tsalka/` — там всё по проекту
2. Или попросите ИИ: *«Найди все документы, связанные с Tsalka»*

### «Что такое _AI_CONTEXT.md?»
Файл-инструкция для ИИ в каждой папке. Объясняет ИИ, что хранится в этой папке и как с ней работать. Вам его редактировать не нужно.

### «Что такое _Drafts?»
Папка для черновиков. Всё, что создаёт ИИ, сначала попадает сюда. Это «песочница» — файлы отсюда не считаются официальными документами.

### «Нужно ли мне знать все номера папок?»
Нет. Достаточно запомнить номера **своих** основных папок. Остальные подскажет ИИ или эта таблица:

## 7. Глоссарий

| Термин | Что это |
|--------|---------|
| **_AI_CONTEXT.md** | Файл-инструкция для ИИ, описывающий назначение папки |
| **_Drafts** | Папка для черновиков (песочница) |
| **YAML frontmatter** | Блок метаданных в начале .md файла (между `---`) |
| **Foam wikilinks** | Ссылки вида `[[FileName]]` для связи между документами |
| **Sandbox Protocol** | Правило: ИИ создаёт файлы ТОЛЬКО в `_Drafts` |
| **Metadata Protocol** | Правило: ИИ добавляет YAML-заголовок с именем модели |
| **Johnny.Decimal** | Система нумерации папок (00-99) |
| **PMBOK** | Стандарт управления проектами (фазы 1-9) |
| **Минцберг** | Теория организационного дизайна (6 частей организации) |
| **Owner** | Ответственный за папку / документ |
| **PII** | Персональные данные (Personal Identifiable Information) |
| **RACI** | Матрица ответственности (Responsible, Accountable, Consulted, Informed) |
